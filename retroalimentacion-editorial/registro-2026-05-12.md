# Registro inicial 2026-05-12

## Contexto

AulaTeX nace como una evolucion local de `Template-Informe`. El proyecto ya no
se entiende solo como una plantilla LaTeX aislada, sino como un entorno
academico para materias, actividades, referencias, plantillas, compilacion,
diagramas TikZ y registro de decisiones.

## Aportes planteados por el usuario

- Crear carpetas de referencias bibliograficas por materias, temas y contextos.
- Guardar documentos conceptuales, libros, archivos `.bib` y materiales de base
  junto a la materia o proyecto que los utiliza.
- Organizar materias como Filosofia del Derecho, Etica y Moral juridica,
  Redaccion en contextos virtuales, materias del IIIEPE y otros trabajos.
- Renombrar y orientar el proyecto como AulaTeX-Academico.
- Mejorar el flujo de compilacion para que funcione con Codex, VS Code LaTeX
  Workshop y TeXstudio.
- Definir salidas predeterminadas para PDFs compilados y exportaciones TikZ.
- Crear una carpeta `Revision` para recibir materiales, plantillas y proyectos
  antes de decidir su integracion final.
- Integrar plantillas LaTeX externas y adaptadas sin obligarlas a depender de un
  unico `main-informe.tex`.
- Dar espacio completo a plantillas como poster y CV aunque todavia no se usen.
- Crear esta metabitacora como memoria historica, registro de propuestas y base
  de lineamientos del proyecto.

## Cambios implementados hasta este punto

- Se organizo `referencias/` por materias, instituciones, proyectos y recursos
  generales.
- Se separo el trabajo academico en `trabajos/`, con espacio para materias,
  instituciones, proyectos y diagramas.
- Se creo `plantillas/latex/` como catalogo de bases originales, adaptaciones
  institucionales y moldes por materia.
- Se integro el catalogo de plantillas de Template-Latex en
  `plantillas/latex/ppizarror/originales/`.
- Se movieron herramientas auxiliares a `herramientas/latex/`.
- Se configuro un flujo de compilacion con `latexmk`, `build/latex` y
  `salidas/pdf`.
- Se agrego exportacion TikZ hacia `salidas/tikz/pdf`,
  `salidas/tikz/svg` y `salidas/tikz/png`.
- Se documento la estructura general en `docs/ESTRUCTURA.md`.
- Se documento el flujo de compilacion en `docs/COMPILACION.md`.
- Se conservo `Revision` como bandeja de entrada local, ignorando su contenido
  pesado en Git salvo los archivos de inventario y lectura.
- Se creo `retroalimentacion-editorial/` para registrar el pensamiento del
  proyecto sobre si mismo.

## Decisiones relevantes

- Las plantillas originales se conservan como fuentes base; los trabajos reales
  deben crearse como copias en `trabajos/`.
- Las plantillas poster, presentacion, articulo, reporte, tesis, controles,
  auxiliares y CV se consideran familias legitimas del catalogo, aunque no todas
  se usen de inmediato.
- `Revision` no es archivo historico final: es una zona de llegada y discusion.
- Las salidas compiladas no deben mezclarse con fuentes academicas.
- La bitacora editorial debe ayudar a tomar decisiones futuras sin depender de
  memoria informal.

## Pendientes vivos

- Definir el primer flujo concreto para una actividad nueva de Etica y Moral
  juridica.
- Crear una convencion fina para actividades que combinan informe y
  presentacion, como Temas Selectos de Matematicas I.
- Evaluar como integrar proyectos Slidev sin mezclarlos de forma forzada con
  documentos LaTeX.
- Mejorar el catalogo de plantillas con descripciones, origen, estado y usos
  recomendados.
- Evolucionar el flujo TikZ hacia conversiones mas automaticas y rastreables.
